## REQUISITO FUNCIONAL

### UX

1. Inputs `#reg-search-cod` y `#reg-search-desc` cambian comportamiento:
   - **Mientras se escribe** (oninput): NO renderizar nada nuevo, solo limpiar el resumen `#reg-resumen-cod` si está visible.
   - **Al presionar Enter** (onkeydown, key === 'Enter'):
     - Si el input tiene valor → push a `state.registros2026Queries` un objeto `{ type: 'cod' | 'desc', value: <trim>, ts: Date.now() }`.
     - Limpiar el input que disparó el Enter.
     - Llamar `renderRegistros2026()`.
     - Si el valor exacto ya existe en la cola (mismo type+value), no duplicar — mover al final (refresh).
2. Input `#reg-filter-patente` mantiene comportamiento oninput actual — es filtro transversal aplicado a TODAS las secciones acumuladas.
3. Checkbox `#reg-show-dif` mantiene comportamiento — toggle global aplicado a todas las secciones.
4. **Nuevo botón** `🗑 Limpiar consultas` en `.registros-actions` (mismo grupo que "Exportar a Excel"):
   - Visible solo cuando `state.registros2026Queries.length > 0`.
   - onclick → vacía `state.registros2026Queries`, limpia inputs cod/desc, llama `renderRegistros2026()`.
5. Si `state.registros2026Queries.length === 0` → mostrar mensaje vacío amigable dentro de `#registros-2026-table` (no romper el layout):
   `<tbody><tr><td colspan="9" style="text-align:center;color:#64748b;padding:24px">Escribe un código o descripción y presiona Enter para agregar una consulta.</td></tr></tbody>`.
   El thead puede quedar oculto en este estado.

### RENDER — secciones acumuladas

Para cada query en `state.registros2026Queries` (en orden de inserción, top→bottom):

1. Filtrar `state.registros2026` por esa query (cod o desc, lowercase includes — misma lógica que hoy).
2. Aplicar **adicionalmente** el filtro PATENTE actual (`#reg-filter-patente`) si tiene valor.
3. Renderizar bloque:
   - `<thead>` con headers azules (igual al actual) — repetidos por sección, como en la imagen.
   - `<tbody>` con las filas filtradas.
   - **Antes** del thead de cada sección (excepto la primera): una fila visual separadora — usar `<tr class="reg-separator"><td colspan="N"></td></tr>` con margen vertical via CSS, O simplemente `<tr><td colspan="N" style="height:18px;border:none;background:transparent"></td></tr>`.
4. El `#reg-resumen-cod` se muestra solo cuando hay exactamente 1 query y es de tipo `cod` (preservar comportamiento útil actual). Para 2+ queries → ocultar.
5. La tabla sigue siendo UNA sola tabla `#registros-2026-table` (no múltiples tablas), respetando el HTML existente.

### EXPORT EXCEL — espejo del visible

`exportRegistros2026Excel()`:

1. Si `state.registros2026Queries.length === 0` → `showToast('Agrega al menos una consulta antes de exportar.', 'error')` y return.
2. Construir `data = [header]` UNA sola vez.
3. Para cada query (en orden): aplicar mismo filtro que el render (query + patente live + showDif toggle), agregar las filas. Entre secciones agregar **una fila vacía** (`Array(cols.length).fill('')`) como separador visual.
4. Mantener todos los estilos actuales (HDR, CEL, anchos, freeze de fila 1, sheet 'REGISTROS', nombre `REGISTROS_2026.xlsx`).
5. NO incluir headers repetidos en el Excel (en pantalla sí ayuda visualmente, en Excel ensucia). Si el usuario quiere, lo puede agregar después — diseño minimalista.

## DETALLES TÉCNICOS

- **Sin librerías nuevas**. Sin frameworks. Sin npm install. Solo XLSX (ya cargado).
- Conservar mismos selectores actuales (`#reg-search-cod`, `#reg-search-desc`, `#reg-filter-patente`, `#reg-show-dif`, `#registros-2026-table`, `#reg-resumen-cod`).
- onkeydown se puede agregar inline en el HTML (`onkeydown="if(event.key==='Enter') regAddQuery('cod', this)"`) o vía addEventListener en una función `_regInitHandlers()` llamada al final del archivo. **Preferir handler inline** para consistencia con el estilo actual del archivo (toda la app usa onclick/oninput inline).
- Si optás por handler inline, crear función helper `regAddQuery(type, inputEl)` que hace el push + limpia input + llama render. Eso evita lógica repetida.
- El `oninput` actual de `#reg-search-cod` y `#reg-search-desc` (`renderRegistros2026()`) **debe eliminarse** — ya no aplica.
