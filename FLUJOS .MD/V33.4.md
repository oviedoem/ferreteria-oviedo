## Tecnologías

- **Firebase Hosting** — servidor estático + CDN (plan Spark gratuito)
- **Firebase Firestore** — base de datos (SDK compat v9.22.0, plan Spark gratuito)
- **Firebase Auth** — autenticación email+contraseña + Google Sign-In (SDK compat v9.22.0)
- **Service Worker** — caché offline + banner de actualización (PWA)
- **jsPDF + jsPDF-autotable** — generación de cotizaciones en PDF
- **Chart.js 4.4.0** — gráficos históricos de cotizaciones (solo panel-admin.html)
- **Cropper.js 1.5.13** — recorte de imagen del banner en panel-admin.html
- **Python + pandas + openpyxl** — procesamiento del catálogo desde ERP
- **Node.js** — script predeploy para actualizar timestamp del SW
- **wa.me links** — integración WhatsApp Business sin costo (sin API de pago)
- **Fuente** — Inter (Google Fonts)
- ~~Firebase Storage~~ — **eliminado en V31** (banner guardado como base64 en Firestore)

## Reglas Firestore — V33.1

Archivo: `firestore.rules`

| Colección | Lectura | Escritura |
|---|---|---|
| `users/{userId}` | `get`: propio UID o admin · `list`: solo admin | `create`: propio UID · `update`: propio (sin campos sensibles) o admin · `delete`: admin |
| `config/{docId}` | `sessionConfig` y `registroControl`: pública · resto: auth | solo admin |
| `cotizaciones` | cualquier auth | `create`: any auth · `update`: propio/admin/vendedor · `delete`: admin |
| `sesiones_activas` | cualquier auth | propio UID o admin |
| `promos` | pública | solo admin |
| `productos` | pública | solo admin |
| `solicitudes_cupon` | propio cliente / admin / vendedor | `create`: any auth · `update/delete`: propio o admin |
| `sesionesLog` | solo admin | cualquier auth |
| `auditLog` | solo admin | cualquier auth |
| `ventas` | cualquier auth | bloqueado (solo script Python con token CLI) |
| `ventasLineas` | solo admin | bloqueado |
| `comandos` | solo admin | solo admin |
| catch-all `/{collection}/{docId}` | bloqueado | bloqueado |

**Helpers:** `autenticado()`, `esAdmin()`, `esVendedor()`

**Campos protegidos en update de `users`:** `role`, `estado`, `registroAprobado` — solo el admin puede modificarlos.

## Monitor de Cuota Firebase — V33.1 / V33.2

Implementado en `panel-admin.html`. Rastrea lecturas/escrituras/deletes del día usando `localStorage`.

### Componentes

| Componente | Descripción |
|---|---|
| **Topbar badge** `#quotaTopBadge` | Muestra lecturas + escrituras del día. 🔥 ok / ⚠️ warn / 🚨 crit. Click abre modal. |
| **Modal detalle** `#quotaModal` | Gauges del día, tabla historial semana/mes, registro manual, reset. |
| **Inline tab-mejoras** `#quotaGaugesInline` | Cards compactas de cuota dentro de la pestaña Mejoras. |

### Funciones clave

| Función | Descripción |
|---|---|
| `quotaTrackRead(n)` / `quotaTrackWrite(n)` | Registrar operaciones (llamar desde cada fetch/write Firestore) |
| `quotaActualizar()` | Refresca badge + inline display |
| `quotaAbrirModal()` / `quotaCerrarModal()` | Abrir/cerrar modal de detalle |
| `quotaTabCambiar(tab)` | Cambiar entre semana / mes / historial |
| `quotaRegistrarManual()` | Registrar operaciones de scripts externos |
| `quotaResetHoy()` | Resetear contadores del día actual |

### Storage localStorage

Clave por día: `fq_YYYY-MM-DD` → `{reads, writes, deletes, log[]}`

### Límites Spark

| Operación | Límite diario |
|---|---|
| Lecturas | 50.000 |
| Escrituras | 20.000 |
| Deletes | 20.000 |

## Flujo de autenticación — V31

### Ruta A — Email + Contraseña

```
signInWithEmailAndPassword(auth, email, pass)
  → verifica estado Firestore users
    → estado='bloqueado'         → error
    → registroAprobado=false     → "esperando aprobación"
    → role no coincide con panel → error + signOut()
    → ok                         → mostrarApp*()
```

### Ruta B — Google Sign-In

```
GoogleAuthProvider → signInWithPopup
  → verifica UID en Firestore users
    → no existe → signOut() + "cuenta no registrada"
    → ok        → mostrarApp*()
```

Google Sign-In NO crea usuarios. El admin debe registrar la cuenta primero.

### Ruta C — Recuperar contraseña

`sendPasswordResetEmail(auth, email)` — disponible en los 3 paneles.

## Pipeline catálogo — ACTUALIZAR_Y_PUBLICAR.bat

```
[0]  descargar_erp.py                 → actualizar.xlsx
       Stock: Reporte_Bodegas_Detalle.asp (HTTP directo, IdBodega 22+13)
       Precios: VisorRS via Playwright (exportReport 'CSV') → CSV → DataFrame
       Fallback precios: CSV local ~/Downloads < 24h
[CONFIG] Visibilidad de precios S/N/Enter → actualizar_config_precios.py (auto: E en 30s)
[1]  actualizar.xlsx → Datos.xlsx     (procesar-actualizacion.py)
[2]  Datos.xlsx → Datos.csv           (xlsx_a_csv.py)
[3]  Datos.csv → Datos.json           (csv_a_json.py)
[4]  firebase deploy --only hosting   (predeploy: update-sw-version.js)
```

> **Nota V33.3:** VisorRS exporta en formato **CSV** (no Excel). Soluciona el error diario
> `Sys.InvalidOperationException: The report or page is being updated` del ReportViewer SSRS.
> El script espera con `wait_for_function()` hasta que `get_reportAreaContentType()` no lanza
> excepción antes de llamar `exportReport('CSV')`. Todos los prompts interactivos del bat
> tienen timeout 30 s con default seguro (E / continuar).

## Service Worker (sw.js)

| Tipo de recurso | Estrategia |
|---|---|
| HTML / navegación | Network-First |
| `.js`, `.css`, `.jpg`, `.png` | Cache-First |
| `Datos.json` | Sin caché (siempre red) |
| Firestore / Firebase APIs | Sin caché |

Actualización detectada automáticamente: `firebase deploy` → `update-sw-version.js` cambia `BUILD_DATE` → nuevo `CACHE_NAME` → banner "Nueva versión disponible".
